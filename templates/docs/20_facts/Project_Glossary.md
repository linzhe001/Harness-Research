## Naming Rules

- Use one canonical term per concept.
- Keep code identifiers, schema keys, config keys, metric keys, and file names in English.
- Use nouns for entities and artifacts.
- Use verbs for commands and scripts.
- Include units or direction in metric names when ambiguity is possible, such
  as `latency_ms`, `accuracy_top1`, or `loss_lower_is_better`.
- Use `run_id`, `experiment_id`, and `artifact_id` consistently.
